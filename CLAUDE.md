# Overview

MCP (Model Context Protocol) server that exposes TailwindPlus UI components to AI assistants. Built with Python and FastMCP, it reads a JSON data file generated by the [TailwindPlus Downloader](https://github.com/BryantDesigns/ts-tailwindplus-downloader) (`>= 1.0.0`), parses it into a SQLite cache, and serves components over stdio.

# Commands

```bash
uv sync                                  # install dependencies
uv run pytest                            # run all tests
uv run pytest tests/test_tailwind_plus.py::TestClassName::test_name  # run a single test
uv run ruff format . && uv run ruff check . --fix  # format and lint (run before every commit)
uv run tailwindplus-mcp-connector --tailwindplus-data /path/to/components.json  # run locally
uv run tailwindplus-mcp-connector --clear-cache   # clear SQLite cache
```

# Architecture

## Key files
- `src/tailwindplus_mcp_connector/__init__.py` — CLI entrypoint (`main()`); parses args, resolves data file path via `--tailwindplus-data` flag or `MCP_TAILWINDPLUS_DATA` env var, instantiates `TailwindPlus`, creates and runs the server
- `src/tailwindplus_mcp_connector/tailwind_plus.py` — `TailwindPlus` class; all business logic: parses JSON, builds/reads SQLite cache, provides all query methods
- `src/tailwindplus_mcp_connector/server.py` — `create_server()` factory; registers `TailwindPlus` methods as FastMCP tools and resources (no decorator pattern — uses `server.tool()` and `server.resource()` calls)

## Data flow
1. On startup, `TailwindPlus.__init__` checks if the SQLite cache is stale (by mtime/size of the JSON file)
2. If stale or missing, rebuilds the cache by traversing the nested JSON tree into a `snippets` table
3. All queries hit SQLite; component names are preloaded into memory as a `set[str]`

## MCP tools exposed
- `list_component_names` — returns sorted list of all dot-path component names
- `search_component_names(search_term)` — case-insensitive partial match
- `get_component_by_full_name(full_name, framework, tailwind_version, mode)` — returns `Component` dataclass
- `get_component_preview_by_full_name(...)` — returns preview HTML string
- `list_tailwindplus_information` — returns metadata from the data file

## MCP resources exposed
- `twplus://{component_full_name}/{framework}/{version}/{mode}` — component code (JSON)
- `twplus://{component_full_name}/{framework}/{version}/{mode}/preview` — preview HTML

## Mode rules (enforced server-side)
- **Application UI** and **Marketing** components: `mode` must be `light`, `dark`, or `system`
- **Ecommerce** components: `mode` must be `none`
- Validated in `_validate_mode()` by checking if `full_name` starts with `"Ecommerce"`

## Component naming
Components use dot-separated hierarchical paths matching the TailwindPlus category structure, e.g. `Application UI.Forms.Input Groups.Label with leading icon`. The `ComponentNotFoundError` includes up to 5 suggestions on lookup failure.

## Cache location
SQLite cache is stored in the platform cache dir (`platformdirs.user_cache_dir("tailwindplus-mcp-connector")`), keyed by a SHA-256 hash of the data file's absolute path.

# Project notes

- Does **not** use FastMCP's decorator methods — all tools and resources are registered imperatively in `server.py`
- Resource handler methods (`get_component_as_resource`, `get_component_preview_as_resource`) are thin string-to-enum adapters that delegate to the tool methods
- Downloader version `>= 1.0.0` is required; validated on cache build via `packaging.version.Version`
